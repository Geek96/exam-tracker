# ADR-002: 使用 MinerU 作为 PDF 内容提取引擎

**日期**: 2026-04-28  
**状态**: 已接受

## 背景

需要将用户上传的课本 PDF 转换为结构化 Markdown，供 AI 读取和分析。

## 决策

使用上海 AI 实验室（SAIL）开发的 MinerU API（`mineru.net`）作为 PDF → Markdown 转换服务。

## 理由

- **学术文档质量高**：专为学术/教材 PDF 优化，正确处理公式、表格、多列布局
- **支持 HTML 输入**：`MinerU-HTML` 模型可处理 HTML 格式的课件
- **免费额度**：有足够的免费配额供个人学生使用
- **API 简洁**：提交 URL → 异步轮询结果，接入成本低

## 限制与应对

| 限制 | 应对方案 |
|---|---|
| 每任务 < 200 页 | 客户端用 `pdf-lib` 分片（阈值 199 页），并行提交 |
| 需公网可访问 URL | 浏览器上传至 `litterbox.catbox.moe` 获取临时 URL |
| 处理耗时（1-5 分钟）| 浮动任务状态栏 + 后台轮询（最长 12 分钟超时） |

## 替代方案（未选择）

- **pdf.js 文字提取**：无法处理扫描版 PDF 和复杂排版
- **ChatGPT Vision**：成本高，无 Markdown 导出
- **Adobe Extract API**：付费，集成复杂
